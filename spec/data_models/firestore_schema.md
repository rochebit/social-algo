# Firestore Schema & Security Rules Specification

This document specifies the Firestore database structure, document schemas, indexing requirements, and security rules for the hybrid AT Protocol Feed Monitor.

---

## 1. Firestore Schema & Collections

We utilize two primary collections in Firestore:
1. `posts`: Represents the filtered feed items matching developer keywords or network parameters.
2. `feedback_logs`: Archives all rating events with metadata to support offline analysis.

### 1.1 `posts` Collection
- **Path:** `/posts/{postId}`
- **Document ID (`postId`):** SHA-256 hash of the post's ATProto URI.

#### Document Schema
```json
{
  "uri": "at://did:plc:rpqw572o3uowvjscsps5u7e6/app.bsky.feed.post/3ks5z3a2jzk2c",
  "cid": "bafyreihymx...",
  "authorDid": "did:plc:rpqw572o3uowvjscsps5u7e6",
  "authorHandle": "devguy.bsky.social",
  "text": "Check out this new ATProto AppView implementation in Rust! https://github.com/... #atproto",
  "createdAt": "2026-07-01T11:45:00.000Z",
  "matchedAt": "2026-07-01T11:45:05.123Z",
  "relevanceScore": 85,
  "relevanceExplanation": "Post mentions ATProto AppView implementation in Rust with a link to source code.",
  "matchRules": ["keyword:atproto", "keyword:appview"],
  "feedback": "positive", 
  "feedbackAt": "2026-07-01T11:50:00.000Z",
  "isDeleted": false,
  "parentContext": {
    "uri": "at://did:plc:anotherdev/app.bsky.feed.post/999",
    "authorHandle": "seniorguy.bsky.social",
    "text": "Has anyone tried building an AppView in Rust yet?"
  },
  "quotedContext": null,
  "facets": [
    {
      "start": 58,
      "end": 79,
      "type": "link",
      "uri": "https://github.com/..."
    },
    {
      "start": 80,
      "end": 89,
      "type": "tag",
      "tag": "atproto"
    }
  ],
  "mediaEmbed": {
    "type": "images",
    "images": [
      {
        "thumbUrl": "https://cdn.bsky.app/img/feed_thumbnail/plain/did:plc:rpqw572o3uowvjscsps5u7e6/bafyimgcid@jpeg",
        "fullsizeUrl": "https://cdn.bsky.app/img/feed_fullsize/plain/did:plc:rpqw572o3uowvjscsps5u7e6/bafyimgcid@jpeg",
        "alt": "Screenshot of build log"
      }
    ],
    "externalLink": null,
    "video": null
  }
}
```

| Field Name | Type | Description |
|---|---|---|
| `uri` | string | The full AT Protocol URI of the post. |
| `cid` | string | The IPFS CID of the post record. |
| `authorDid` | string | The decentralized identifier of the post author. |
| `authorHandle` | string | The current handle of the post author at matching time. |
| `text` | string | The textual content of the post. |
| `createdAt` | string (ISO-8601 UTC) | The creation timestamp defined by the post author. |
| `matchedAt` | string (ISO-8601 UTC) | The ingestion timestamp when the home server processed the post. |
| `relevanceScore` | number | A score between `0` and `100` generated by the LLM classification step. |
| `relevanceExplanation`| string | The natural language reasoning provided by the LLM. |
| `matchRules` | array of strings | List of regex/rule patterns triggered prior to LLM evaluation. |
| **`feedback`** | string or null | Whitelisted user ratings: **`"negative"`**, **`"neutral"`**, **`"positive"`**, **`"extra_positive"`**, or `null`. |
| `feedbackAt` | string (ISO-8601 UTC) or null | Timestamp of when the user submitted the feedback. |
| `isDeleted` | boolean | Set to `true` if the post is deleted on Bluesky. Defaults to `false`. |
| `parentContext` | object or null | Context of the post being replied to. Null if not a reply. |
| `quotedContext` | object or null | Context of the post quoted/embedded. Null if not a quote. |
| `facets` | array of objects | Extracted rich-text features. Each contains byte range (`start`, `end`) and type specifics (`uri` or `tag`). |
| `mediaEmbed` | object | Media assets associated with the post. Contains sub-objects `images` (array), `externalLink` (object), and `video` (object). |

### 1.2 `feedback_logs` Collection
- **Path:** `/feedback_logs/{feedbackId}`
- **Document ID (`feedbackId`):** Automatically generated UUID / Firestore ID.
- **Purpose:** An append-only log of feedback events. Even if a post is deleted or feedback is updated, this logs the historical feedback data for model assessment.

#### Document Schema
```json
{
  "postUri": "at://did:plc:rpqw572o3uowvjscsps5u7e6/app.bsky.feed.post/3ks5z3a2jzk2c",
  "postText": "Check out this new ATProto AppView implementation in Rust! https://github.com/... #atproto",
  "authorDid": "did:plc:rpqw572o3uowvjscsps5u7e6",
  "relevanceScore": 85,
  "feedbackValue": "positive",
  "timestamp": "2026-07-01T11:50:00.000Z"
}
```

## 2. Required Indexes

To query the feed efficiently on the client (e.g., retrieving the latest unclassified or unrated posts), the following indexes must be provisioned in Firebase:

### Single-field Indexes
- `matchedAt` (Descending): Used to display the feed in reverse chronological order.

### Composite Indexes
- Collection: `posts`
  - Fields: `isDeleted` (Ascending), `feedback` (Ascending), `matchedAt` (Descending)
  - Purpose: Retrieve all active posts (`isDeleted == false`) that do not yet have feedback (`feedback == null`) sorted by descending matched time.

## 3. Firebase Security Rules (`firestore.rules`)

To secure the database, we restrict read/write access exclusively to your verified Google email address. No public reads or writes are allowed.

```javascript
rules_version = '2';

service cloud.firestore {
  match /databases/{database}/documents {
    
    // Helper function to check if the user is signed in and matches the owner email
    function isOwner() {
      return request.auth != null 
        && request.auth.token.email_verified == true
        && request.auth.token.email == "OWNER_EMAIL_PLACEHOLDER";
    }

    // Rule for matching posts
    match /posts/{postId} {
      allow read, write: if isOwner();
    }

    // Rule for logging feedback
    match /feedback_logs/{feedbackId} {
      allow read, write: if isOwner();
    }
  }
}
```

> [!IMPORTANT]
> The developer/agent implementing this deployment must replace `OWNER_EMAIL_PLACEHOLDER` with the owner's actual Google Account email address during the Firebase deploy step.
